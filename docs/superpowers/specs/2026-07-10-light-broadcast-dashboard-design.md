# 浅色世界杯转播台 UI 设计

状态：用户已选择方案 A，并要求执行。

## 目标

把现有仪表盘改成适合每天反复查看的浅色世界杯转播台。页面继续保留完整赛程、预测和复盘信息，但将比分、比赛状态和开赛时间提升为第一视觉层级，减少白色卡片堆叠和重复说明。

## 设计方向

- 用途：快速判断今天有哪些比赛、比赛处于什么状态、当前或最终比分是什么，以及赛前预测是否命中。
- 受众：需要频繁刷新赛程、预测与赛后结果的用户。
- 基调：浅色、专业、紧凑、具有体育转播感。
- 记忆点：深海军蓝比分牌配合左侧状态色轨；比分始终是比赛卡片中最醒目的元素。
- 约束：沿用 Next.js、Tailwind、Lucide 图标和现有路由；不增加视觉依赖，不改变数据接口与模型逻辑。

## 页面信息层级

1. 顶部导航保持吸顶，但降低视觉重量，使用白色半透明表面和清晰的绿色激活态。
2. 首屏介绍区改为紧凑的赛事控制台，显示北京时间、数据同步状态、今日场次、进行中数量和最近复盘数量。
3. 今日比赛紧跟控制台出现，保证桌面首屏能够看到至少一排比赛卡片。
4. 明日赛程和后续淘汰赛按时间顺序排列。
5. 进行中比赛使用实时红色状态，结束比赛保留成功或失败复盘入口。
6. 历史小组赛降低视觉优先级，放在页面底部，并采用更安静的标题和卡片状态。

## 视觉系统

- 页面底色：暖灰绿色 `#f3f6f2`，保留极淡球场网格。
- 主表面：白色 `#ffffff`，边框使用低对比冷灰。
- 主文字：深海军蓝 `#0a1a2b`。
- 品牌绿色：`#11875d`，用于激活导航、强队和积极状态。
- 数据蓝：`#2563eb`，用于赛前预测和主胜概率。
- 实时红：`#dc3345`，仅用于进行中、失败和需要立即关注的状态。
- 提醒琥珀：`#d98b16`，用于冷门和数据不完整提示。
- 阴影保持柔和，避免玻璃卡片层层嵌套。

## 赛事控制台

- 左侧显示“2026 世界杯赛事中心”和当前北京时间。
- 中间显示数据同步状态；正常时为绿色圆点，后端不可用时为琥珀提示。
- 右侧使用四个紧凑指标：今日、明日、进行中、淘汰赛已结束。
- 删除重复的口径介绍卡，仅保留一行“90 分钟口径，不含加时与点球”。

## 比赛卡片

- 卡片左侧增加 4 像素状态轨：未开始为蓝色，进行中为红色，已结束为深绿或灰色。
- 顶部一行显示赛事阶段和状态徽标；状态固定在右上角，不能被长赛事名称挤走。
- 中间使用三列布局：主队、比分牌、客队。比分牌使用深海军蓝底色和 44 至 52 像素数字。
- 未开始比赛的比分牌显示开赛时间，不使用容易误解的 `0-0`；赛前预测比分单独显示为“首选 2-1”。
- 进行中和已结束比赛显示真实比分，并在比分下方显示分钟或“90 分钟”。
- 强队提示降为辅助信息，避免与真实比分争抢注意力。
- 底部概率条保持三行，但缩短标签并提高数字对齐精度。
- 赛后成功使用绿色结果条，失败使用红色结果条，并显示一条可扫描的主要原因。

## 响应式行为

- 大于等于 1280 像素：三列比赛卡片。
- 768 至 1279 像素：两列比赛卡片。
- 小于 768 像素：单列卡片，导航只显示图标，比分数字缩小但不低于 38 像素。
- 长球队名允许两行，不使用破坏含义的省略文本。
- 状态徽标、比分牌和开赛时间在所有宽度下保持稳定位置。

## 交互与动效

- 卡片悬停仅上移 2 像素并增强阴影。
- 进行中状态使用低频呼吸圆点，不让整张卡闪烁。
- 导航切换和卡片悬停动画控制在 160 至 220 毫秒。
- 尊重 `prefers-reduced-motion`，关闭非必要动效。

## 数据与错误状态

- 继续使用现有比赛、预测和复盘接口，不修改字段含义。
- 后端不可用时，赛事控制台显示单一错误提示；各比赛区不重复展示同一错误。
- 空比赛区使用紧凑空状态，不占用整屏高度。
- 未提供预测时仍显示球队、时间和比赛状态，不显示虚构概率。

## 验证标准

- 桌面首屏直接看到今日比赛卡片，比分和状态为最高视觉层级。
- 未开始比赛不再把 `0-0` 表现为真实比分。
- 进行中、已结束和未开始状态无需阅读正文即可区分。
- 390 像素手机宽度无水平滚动、球队名和状态不重叠。
- 键盘焦点可见，颜色对比满足常规可读性要求。
- `npm run lint`、`npm run build` 和相关前端测试通过。
- 浏览器检查 `/dashboard`、比赛详情跳转和移动端布局。

## 非目标

- 不修改比赛数据、北京时间分组、模型概率或赛后校准逻辑。
- 不在本轮重做分析页、模拟页和三串一业务流程。
- 不使用大幅背景图片、球队徽标抓取或新字体下载。
