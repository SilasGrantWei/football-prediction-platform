# 90 分钟与整场比分并列展示设计

日期：2026-07-12
状态：方案 A 已实现并通过自动化、运行态和浏览器验收

## 目标

保留现有 `homeScore` / `awayScore` 的 90 分钟口径，不改变预测、命中率和赛后复盘逻辑；另外接入并展示官方数据源返回的整场比赛比分。加时赛进球属于整场比分，点球大战比分必须单独展示，不能与场上进球相加。

以 `qf-099` 挪威对英格兰为验收样例：

- 90 分钟比分：`1-1`
- 整场比分：`1-2`
- 结果说明：`加时后`

## 已确认的比分语义

### 现有字段（保持不变）

- `homeScore` / `awayScore`：90 分钟常规时间加伤停补时比分。
- 预测结果、胜平负方向、Top3 命中率、Brier、LogLoss 和赛后复盘继续只使用这两个字段。
- 不允许用整场比分覆盖这两个字段。

### 新增字段

- `fullMatchHomeScore` / `fullMatchAwayScore`：比赛结束时的场上累计比分；包含加时赛进球，不包含点球大战罚入数。
- `penaltyShootoutHomeScore` / `penaltyShootoutAwayScore`：点球大战罚入数；仅在数据源明确返回时保存。
- `resultDecision`：`regulation | extra_time | penalties`，说明比赛是在 90 分钟、加时赛还是点球大战后决出。

以上字段均为可空字段。缺少官方返回时不显示、不猜测，尤其不能根据 `winnerTeamId` 反推一个比分。

## 数据流

1. ESPN/FIFA 赛果适配器先保存记分牌返回的最终场上比分。
2. 若比赛经过加时或点球，继续从 summary 的分节比分提取前两节之和，写回现有 90 分钟字段。
3. 点球大战数字只从来源的独立 shootout 字段读取。
4. 同步服务处理主客场反转时，90 分钟、整场和点球三组比分必须一起反转。
5. 数据库和演示模式持久化三组比分与决胜方式，重启后不能丢失。
6. API 在 Match 对象中返回新增字段；旧客户端因字段可选而保持兼容。

## 存储设计

新增数据库迁移 `005_full_match_score`，为 `matches` 增加：

- `full_match_home_score INTEGER NULL`
- `full_match_away_score INTEGER NULL`
- `penalty_shootout_home_score INTEGER NULL`
- `penalty_shootout_away_score INTEGER NULL`
- `result_decision TEXT NULL`，约束值为 `regulation`、`extra_time`、`penalties`

演示模式的 `DemoMatchStateSnapshot` 同步增加同名可选字段，并保持对旧 JSON 快照的向后兼容。

## 页面设计

### 比赛详情页

中央深色大比分保持现状，继续显示 `1-1`；其说明固定明确为“90分钟比分”。

在大比分下方新增浅色结果条：

- 加时赛：`整场比分 1-2 · 加时后`
- 点球大战：`整场比分 1-1 · 点球 5-4`
- 90 分钟直接结束：`整场比分 2-0 · 90分钟结束`

新增结果条只在比赛已结束且整场比分已被真实来源验证时出现。字段缺失时不显示“0-0”占位。

### 总览比赛卡片

卡片主记分牌继续使用 90 分钟比分。在已结束卡片的记分牌下方增加一行较小的整场结果；若有点球则再附点球比分。这样总览和详情页口径一致，但不会抢占主比分视觉层级。

## 错误处理与真实性规则

- 90 分钟 summary 查询失败：保留已有 90 分钟真值，不用整场比分覆盖。
- 整场比分缺失：保留 90 分钟显示，隐藏整场结果条。
- 点球字段缺失：只显示整场场上比分和“点球后”状态，不伪造点球数字。
- 来源返回负数、非数字或只返回一方比分：整组字段丢弃并记录同步警告。
- 已结束比赛的赛前预测快照不回写、不重算。

## 测试与验收

### API/数据层

- 加时赛样本：记分牌最终 `1-2`，summary 前两节 `1-1`；断言两组比分同时保留。
- 点球样本：断言 90 分钟、整场场上比分和点球比分彼此独立。
- 普通完赛样本：90 分钟和整场比分可以相同。
- 主客场反转样本：三组比分均正确反转。
- 演示快照重启测试：新增字段完整恢复。
- 数据库迁移测试：字段和约束只应用一次。

### 前端

- 详情页展示辅助函数测试三种情况：普通完赛、加时完赛、点球完赛。
- 整场字段缺失时不显示假比分。
- 现有 90 分钟标题、比分和预测复盘断言保持不变。

### 端到端验收

- `GET /api/matches/qf-099` 返回 `homeScore=1`、`awayScore=1`、`fullMatchHomeScore=1`、`fullMatchAwayScore=2`、`resultDecision=extra_time`。
- `/match/qf-099` 同时可见“90分钟比分 1-1”和“整场比分 1-2 · 加时后”。
- 自动同步与电脑重启后结果仍然存在。

## 非目标

- 不新增整场比分预测模型。
- 不把加时或点球结果写入 90 分钟预测命中率。
- 不根据球队强弱、胜者字段或事件数量推断缺失比分。
