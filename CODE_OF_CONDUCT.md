# Code of Conduct

## Our standards

This project should be a practical, respectful place to improve open-source
sports intelligence tooling.

Expected behavior:

- Be direct, constructive, and specific.
- Focus criticism on code, data, tests, documentation, and product behavior.
- Respect different levels of experience and different technical backgrounds.
- Avoid harassment, insults, threats, sexualized language, and discriminatory comments.
- Do not publish private information, credentials, customer data, or non-public provider data.

## Scope

This code of conduct applies to project repositories, issues, pull requests,
discussions, release notes, and project communication channels.

## Enforcement

Maintainers may edit, hide, or remove comments and contributions that violate
these standards. Maintainers may also limit participation for repeated or
severe violations.

Report conduct concerns through the maintainer contact listed in `SUPPORT.md`
or through GitHub's private reporting tools after the repository is published.
